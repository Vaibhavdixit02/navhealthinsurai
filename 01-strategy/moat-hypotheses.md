# Moat hypotheses

**Last updated:** 2026-04-02

| # | Hypothesis | Source | Test | Evidence so far |
|---|------------|--------|------|-----------------|
| 1 | **Insurance integration as moat.** The AI triage/navigation space is crowded and well-funded, but none of them have insurance integration. "Our moat isn't the AI model — it's the insurance integration that these companies don't have." | Email: "Open questions", 2026-03-09, Vaibhav | | |
| 2 | **Proprietary data flywheel.** Every triage interaction, provider recommendation, and outcome creates proprietary data. At ~100K examples (12–24 months), a fine-tuned model starts outperforming generic commercial LLMs for our specific use case. | Email: "Open questions", 2026-03-09, Vaibhav | | |
| 3 | **Captive roll-up solves cold-start.** Acquiring captive operators solves the two hardest problems simultaneously: cold-start membership and cold-start data. 5–7 operators → ~40K covered lives, ~$300M+ managed premium, ~$12M+ annual fee revenue on day one. | Email: "Open questions", 2026-03-09, Vaibhav | | Manav aligned on concept; skeptical of pace ("buying 5 assets in 18 months seems like a disaster"). *(Email: "Open questions", 2026-03-09, Manav)* |
| 4 | **Nav layer as distribution.** Being broker + nav layer together is an easier/higher-likelihood way to acquire customers vs. a pure nav layer. Also generates more interesting data for actuarial decisions. | WhatsApp, 2026-03-08, Vaibhav | | |
| 5 | **Consumer front door → steerage → risk-bearing.** Owning navigation provides the most important component for predicting healthcare spend, enabling individual-level underwriting. | V1 Memo, p.13–14 | | |
| 6 | **Vertical integration.** At scale, bringing together payor + intermediary navigation/consumer front door + high-performing providers creates significant leverage in structuring reimbursement models. Long-term comparable: a tech-enabled Optum or Kaiser Permanente. | V1 Memo, p.3 | | |
